Classes VS IDs Blog

Classes

Classes are used when you want to standardise and apply formatting and styles to multiple elements at once.

Pros

  • Can apply to multiple elements
  • Can apply to multiple types of HTML element (eg p, div, h1, etc)
  • Can apply multiple classes to a single HTML element
  • Applies styles and formatting in order, so can use a hierarchy to overwrite older styles

Cons

  • Can lead to higher complexity in large websites
  • Can be over-used
  • Common class names may get used accidentally or overridden by something else within your code, causing bugs

IDs

IDs are used when you want to apply something to one unique HTML element.

Pros

  • Allows you to reference a single HTML element from anywhere in your website (including other pages)
  • Allows bookmarking of a place in your code
    • Allows you to style or select a specific element via CSS and JavaScript

Cons

  • ID is case-sensitive
  • Each ID can only be used once in your HTML Document
